Jitter varies hdbscan_min_samples slightly across the M replicates so they produce different agreement scores (cuML HDBSCAN on GPU has no random seed — identical inputs can yield identical clusters). For base=20 with M=3 and jitter=1, runs use [20, 21, 19]. Set jitter=0 to run true replicates (may yield identical scores). Jitter keeps the t-SNE+RF cache valid, so all M runs still share one cached embedding.
